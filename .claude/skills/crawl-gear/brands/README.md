# 브랜드 크롤링 메모장 (brands/)

브랜드마다 **사이트 링크·플랫폼·주의사항·무게 출처·한글화 방식**을 적어둔 메모장.
같은 브랜드를 다시 크롤링할 때, 크롤 전에 이 파일을 먼저 읽고 주의사항을 챙긴다.

## 리프레시(신규 추가) 워크플로우
1. main 에서 `crawl/refresh-<날짜>` 브랜치 생성
2. 각 브랜드: 이 메모 읽기 → 크롤러 재실행 → 새 결과 생성
3. **Firestore 의 기존 제품과 대조**(company 로 조회) → 없던 것만 신규
4. 신규만 push (이미지 재업로드 최소화, 유저 데이터 보존)
5. "이번 리프레시: 브랜드별 +N개" 리포트 → 메모의 '마지막 크롤' 갱신 → 커밋 → PR → main 머지

> 신규 판별은 **Firestore 가 기준선**이므로 제품목록 파일(out/*-FINAL.json)을 따로 관리할 필요는 없다.
> 단 **크롤러 코드와 이 메모는 반드시 커밋**해 둬야 다음에 재현·주의가 가능하다.
