# 블랙다이아몬드 (black-diamond)

- **공식**: https://blackdiamondequipment.co.kr (고도몰5, 리스팅·상세 전부 CSR — puppeteer 필수)
- 크롤러: `sites/black-diamond.js`
- 마지막 크롤: 2026-07, 767개 상품 → 옵션 확장 후 **4287행**(품절 사이즈도 포함 — 17번 참고,
  MEN'S/WOMEN'S 의류 + EQUIPMENT 클라이밍·배낭·조명·텐트·스키·풋웨어·글러브). **무게
  3792/4287행(88.5%) 확보** — gloves 99.7%, backpack 97.0%, trekking_pole 96.5%, clothing
  95.9%, lighting 91.9%, tent 88.9%, gaiter 85.7%(아래 15번 참고 — 남는 0g는 그라운드클로스/
  폴 케이스 등 액세서리로 사이트에 무게 표기 자체가 없는 걸 개별 확인함. 진짜 제품은 다 잡힘).

## ⚠️ 주의사항

1. **리스팅·상세 전부 CSR(Vue/jQuery 조합), curl로는 빈 셸만 나온다.** 동일 세션 쿠키로
   `fetch(location.href)`를 해도 상품 데이터가 없다 — 진짜로 클라이언트에서 렌더링하므로
   puppeteer 필수(list/detail 둘 다).
2. **`.goods_description_images` div는 초기 렌더에도 빈 채로 이미 DOM에 존재**해서
   `waitForSelector`만으로는 콘텐츠가 채워지길 못 기다린다 — `waitForFunction`으로 텍스트
   길이가 실제로 늘어날 때까지 기다려야 한다. 이걸 놓치면 무게/소재가 간헐적으로 0/빈값이 됨
   (레이스 컨디션이라 크롤할 때마다 실패하는 상품이 달라짐).
3. **무게/소재는 두 가지 템플릿이 섞여 있다**(상품정보제공고시 표는 전부 "상세설명참조"
   플레이스홀더뿐이라 못 씀):
   - 해외 오리지널 컨텐츠(의류 등): "TECH SPECS ... MATERIALS `<설명>` WEIGHT `<N>` g"
   - 국내 자체 제작(하드웨어 일부): "TECH SPECS 소재 `<설명>` ... 무게: `<N>` g"
   - `parseMaterial`은 "상품 정보 제공 고시" 이전 구간(TECH SPECS)으로만 검색 범위를 한정한다
     — 안 그러면 고시 표 안의 "재질" 라벨이 뒤에 오는 무관한 텍스트(제조자/원산지 등)까지
     통째로 삼킨다.
4. **색상별 이미지가 라벨과 함께 깔끔하게 있다** (AMG 케이스와 다름) — `.bdia_color_option
   li > img[alt=색상명]`이 `onclick="gcsImgView('/data/goods/....jpg', ..._thumb.jpg')"`로
   그 색상 전용 이미지 URL을 직접 갖고 있음. `img.alt`가 곧 색상명이라 순서 추측/HSV 분석
   불필요. 단, 그 URL이 **도메인 없는 절대경로**(`/data/goods/...`)라 `location.origin`을
   붙여야 한다 — 안 붙이면 imageUrl이 상대경로로 저장되는 버그(실제로 겪음, 49/55행 발생).
5. **"One"/"ONE"은 실제 색상이 아니라 옵션 자체가 없을 때 고도몰이 붙이는 기본 라벨**
   (사이즈의 "ONE 사이즈"와 같은 성격) — 색상 select/스와치에 "One"만 있으면 색상 없음으로
   처리해야 한다. 안 그러면 "오느이"처럼 의미 없는 음역이 colorKorean에 들어간다.
6. **색상명이 이미 영문**(브랜드가 국제 브랜드라 Basalt/Creek Blue/Octane 등)이라 별도 영→한
   사전이 필요없는 대신, colorKorean은 발음 음역으로 생성한다. 순수 색상명 사전
   (`KNOWN_COLOR_KO`)에 없는 마감/수식어(Polished, Light, Envy, Ultra 등)는 한 글자씩
   이어붙이면 심하게 깨져서("Polished"→"프오을이스흐이드") 모음 기준으로 음절을 묶는
   `syllabicKorean` 폴백을 씀 — 그래도 완벽하지 않으니 새로 보이는 단어는 사전에 추가할 것.
7. **`slugify`에서 괄호를 지우면 안 된다**(AMG 세션과 동일 함정) — "퍼스트라이트 3P 그라운드
   클로쓰"와 "...클로쓰(K)"처럼 괄호가 서로 다른 상품의 유일한 구분자인 경우가 있다.
8. **이름이 완전히 같은 서로 다른 물리 상품(SKU)이 실제로 존재한다** — 예: "트랜스퍼 쇼블"이라는
   이름의 두 SKU(productNo 7085, 9347)가 "Hyper Red" 색상 하나를 공유. 이런 경우 groupId+
   color+size가 충돌해 push 시 하나가 다른 하나를 덮어쓴다. `resolveGroupIdCollisions`가
   크롤 전체 결과를 모은 뒤 충돌하는 조합에만 productNo를 접미사로 붙여 구분한다.
9. **대량 크롤 중 간헐적으로 실제 서버 503("Server Unavailable") 페이지가 온다** — puppeteer의
   `page.goto`는 HTTP 에러 상태여도 reject하지 않아 그대로 상품명("503 - Server
   Unavailable")으로 저장돼버린다. `fetchDetail`이 결과 상품명을 에러 패턴으로 검사해 최대
   3회 재시도한다. 그래도 안 되면(실제 겪음, 1건) 해당 카테고리만 다시 `--categories=`로
   재크롤하거나 브라우저로 직접 확인 후 수동 패치.
10. **SPORTS(0001)/COLLECTION(0007)/OUTLET(0006) 최상위 그룹은 크롤 대상에서 제외**했다 —
    EQUIPMENT/MEN'S/WOMEN'S 하위 카테고리와 상품이 겹치는 재분류·재노출 뷰라서 중복 크롤을
    막기 위함. NEW/BEST/전체보기 같은 배너성 서브카테고리도 마찬가지로 제외.
11. **카테고리 코드는 12자리**(`category=c` + 12자리)이고 `code=`(짧은 형태, 예: `0003`)로
    접속하면 자동으로 긴 형태로 리다이렉트된다 — 크롤러는 처음부터 긴 형태를 쓴다.
12. **페이지네이션 마크업이 없다** — `page=N`을 계속 늘려서 카드가 0개 나오면 멈추는 방식.
    한 카테고리 최대 30페이지까지만 순회(안전장치).
13. **name(영문) 필드에 한글이 섞이는 초기 버그**(리뷰 중 발견) — 국제 브랜드라 og:title을
    그대로 name에 넣었더니 모델명이 한글 표기뿐인 상품(예: "스트라타라인 스트레치 쉘")이
    2839/2851행(99.6%)에서 name에 한글이 그대로 남았다. `romanizeName`(제로그램/AMG와 동일
    RR 로마자 음역)으로 한글 토큰만 음역하고 영문/숫자(MENS, C4 등)는 그대로 두도록 수정.
    원래 영문 모델명(예: Strataline)을 정확히 복원하진 못하고 발음 음역("Seuteuratarain")
    수준 — 알려진 한계, AMG 세션과 동일.
14. **`specs.material`을 카테고리 무관하게 채우면 스키마 위반 위험** — lighting은 material
    필드가 아예 없고(type/maxBrightness/batteryType 등), tent/shelter는 innerMaterial/
    flyMaterial/poleMaterial로 나뉘어 있다. `MATERIAL_CATEGORIES` 화이트리스트(clothing/etc/
    gloves/backpack/trekking_pole/gaiter)로 제한했다 — 리뷰 시점엔 lighting/tent/shelter에서
    실제로 매치된 값이 없어 기존 데이터엔 영향 없었지만, 재크롤 시 값이 잡히면 스키마를 어길
    뻔한 잠재 버그였다.
15. **텐트/배낭/게이터/트레킹폴은 무게 라벨이 "WEIGHT"/"무게"가 아니라 "중량"·"평균 중량"·
    "개당 무게"** 인 경우가 많고, **사이즈별로 값이 다른 대괄호 표**로 오는 경우가 대부분이라
    (사용자가 카테고리별 수집 수량을 물어봐서 발견 — 최초 크롤 때 45~55%대였던 원인):
    - "WEIGHT [S] 43 g [M] 48 g"(게이터), "WEIGHT [S/M] 1.80 kg [M/L] 1.80 kg"(배낭),
      "개당 무게 [95-110 cm] 154 g[110-125 cm] 168 g"(트레킹폴 길이), "평균 중량 3.2 kg"
      (텐트, 대괄호 없는 단일값)처럼 패턴이 다 다르다.
    - `parseWeightTable`이 `[라벨] 값단위` 쌍을 전부 뽑고, 그 상품의 실제 사이즈 select 값과
      매칭해서(정규화 후 정확매치 → 안 되면 범위 끝 숫자 매치 폴백) **행(사이즈)마다 다른
      무게**를 넣는다 — AMG 세션의 "컬럼 인덱스가 아니라 라벨로 매핑" 원칙과 동일. 라벨/
      사이즈select 표기가 사이트에서도 제각각이라("[S/M]" ↔ select "S_M", "[105cm]" ↔
      select "105") 정규화(공백·슬래시·언더스코어 통일, "cm" 접미사 제거)가 필요했다.
    - 대괄호 표가 없으면 `parseWeightSingle`로 폴백(라벨 여러 개 지원: WEIGHT/무게/중량/
      평균 중량/개당 무게).
    - **사이즈 select가 상세설명 텍스트와 별도 타이밍으로 채워지는 레이스 컨디션이 또 있었다**
      (설명 텍스트는 채워졌는데 사이즈 옵션이 플레이스홀더 1개뿐이라 사이즈가 통째로 빈값이
      되는 버그, 실제 겪음) — `waitForFunction`에 사이즈 select populate 조건도 같이 걸었다.
    - **사용자가 "배낭/텐트는 무게가 무조건 있어야 한다"고 재확인 요청 → 실제로 더 있었다**
      (57~74%였던 게 82~96%까지 개선됨). 추가로 찾은 패턴/함정:
      - 오타 "**WHIGHT**"(정식 WEIGHT 아님)가 실제 상품 페이지에 있음 — 정규식에 `W[EH]IGHT`로
        같이 잡음.
      - "WEIGHT PER PAIR 534g"처럼 라벨과 숫자 사이에 수식어가 끼는 경우 — `(?:PER\s*(?:PAIR|
        UNIT))?` 허용.
      - "WEIGHT 720g(unisex) / 668g(womens)"처럼 성별로 값이 갈리는 경우 — 상품명에 WOMENS
        포함 여부로 골라 씀.
      - 대괄호 표 라벨이 실제 사이즈 select와 아예 다른 체계인 경우("[M]" 하나뿐인데 select는
        SML/MED/LRG) — 값이 **하나뿐이면 사이즈 무관하게 그 상품 전체의 대표값**으로 쓰고,
        값이 여러 개인데 사이즈 select 자체가 없는 상품(트레킹폴 일부)은 첫 값을 근사치로 씀
        (정확한 사이즈 매핑은 불가하지만 0보다 낫다고 판단).
      - SML/MED/LRG처럼 사이즈가 단어 축약형인데 표 라벨은 S/M/L 한 글자뿐이면 첫 글자로 매칭.
    - 그래도 남는 잔여 저조 구간(tent 61%)은 개별 확인 결과 그라운드클로스/폴 케이스/
      스페어파츠 같은 액세서리 상품 페이지에 무게 텍스트 자체가 없는 진짜 데이터 부재.
16. **같은 상품이 사이트에서 카테고리 두 곳에 걸쳐 있다**(예: 그라운드클로스가 "스페어파츠/
    ACC"와 "텐트/비비" 양쪽, 월홀러 백이 "빅월"과 "클라이밍배낭" 양쪽) — 특정 카테고리만
    재크롤해서 기존 결과와 병합할 때 `(productNo, color, size)` 기준으로 중복이 생긴다.
    병합 시 `etc`(범용 catch-all)보다 더 구체적인 카테고리를 우선해서 하나만 남긴다.
17. **사이즈 select의 "품절" 표시를 이유로 그 사이즈를 통째로 제외하면 안 된다**(사용자가
    명시적으로 지적 — 과거 소유 기록 등을 위해 지금 안 팔아도 사이즈 자체는 완전 수집해야
    함). 처음엔 `!/품절/.test(t)`로 걸러서 "XL (품절)"이 있는 상품은 XL 자체가 빠졌었다
    (예: 페이디드 티 4색×4사이즈=16이어야 하는데 12행만 나옴). "(품절)" 표시 문자열만
    제거하고 사이즈 값 자체는 그대로 담도록 수정 — 옵션 확장 행 수가 2881→4287로 늘었다.
    색상 스와치(`.bdia_color_option`)는 표본 확인 결과 품절 마킹이 select 옵션 수와 항상
    일치해 이 문제가 없었다(사이즈만 품절 마킹이 흔한 것으로 보임).
