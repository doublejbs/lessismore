# 엑스패드 (exped)

- **한국 공식**: https://exped.co.kr  (Cafe24, 매트·텐트·침낭·백팩 등 기어 브랜드)
- **국제 공식**: https://www.exped.com/en  (커스텀 CMS, 제품 페이지에 구조화 스펙 — 1차 무게 소스)
- 크롤러: `exped.py`(KR 크롤 + EN 스펙 음역매칭), EN 스펙 캐시 `out/exped-en-cache.json`
- 후처리: `exped-weight-ocr.py`(무게 OCR 폴백), `exped-color-ocr.py`(색상 OCR)
- 마지막 크롤: 2026-07, 335행(제품 164). 무게 260/335 — 매트 138/138·침낭 33/33·텐트 31/33.

## ⚠️ 주의사항

### 크롤 & EN 매칭
1. **KR 카테고리 8개**(텐트24·매트25·백팩26·스토리지27·침낭28·베개42·해먹43·액세서리44) 모두 순회.
   풋프린트→tent_acc, 레인커버→backpack_cover, 스트랩/카라비너 등→etc 로 보정.
2. **사이즈 옵션(M/LW/MW 등)은 상세 옵션 select 에 있다.** 첫 옵션이 "선택하세요" 플레이스홀더 →
   거기서 멈추지 말고 skip, "SHIPPING TO" 나오면 종료. **옵션에 색상이 섞임**(예 'S-M-블랙') →
   색상 토큰 분리해서 color 로, 나머지를 size 로.
3. **1차 무게 = 국제 사이트**(exped.com/en) 제품 페이지의 `<h4>라벨</h4><div>값</div>` 블록:
   - 매트/침낭/백팩: `Weight`(사이즈별 "M: 1010 g | LW: 1415 g" — KR 사이즈옵션과 매칭).
   - **텐트: `Max. Weight`(패킹) 우선, `Min. Weight`(최소)는 폴백.** (라벨이 'Weight' 아님 주의!)
   - R-Value, Temperature(℃), Volume(L), Thickness, Person Capacity 도 같은 구조.
4. **KR↔EN 매칭 = 음역(MODEL_EN) + 숫자.** 버사→versa, 씸→sim, 플렉스→flex, 울트라→ultra, 테라→terra,
   두라/듀라→dura, 토렌트→torrent, 스카이라인→skyline 등. **텐트 인용수는 EN 이 로마숫자**(3→iii) →
   arabic/roman 둘 다 시도. 매칭 안 되면 EN 무게 없음(KR 전용 모델·음역 미등재) → 아래 OCR 폴백.
5. num_key 에 사이즈(M/LW)를 넣지 말 것 — 무게는 wmap[size] 로 별도 매칭.

### 무게 OCR 폴백 (`exped-weight-ocr.py`) — EN 매칭 실패분 대상
KR 전용/미매칭 매트·침낭·텐트는 무게가 **KR 상세 이미지(`/web/upload/NNEditor/`)의 스펙 표** 안에 있다.
macOS Vision OCR(~1500px 타일) 후 표 파싱. **표가 6형식**이라 모두 지원해야 한다:
6. - **(A) 행형(침낭)**: 사이즈로 시작 `M 220cm … 300g`.
   - **(B) 매트 헤더형**: `M 사이즈`(또는 단독 `MW`/`LW`/`LXW`) 헤더 + 다음 값라인 `…595 g`. 단독 헤더는 2글자+ 사이즈만(단일 S/M/L 은 노이즈).
   - **(C) 컬럼형**: `사이즈 MW LW LXW` 헤더행 + `무게 2040g 2200g 2575g` 행. OCR 이 `사이즈`→`사 이 즈`, `무게`→`무낵 게` 로 깨져도 대응(`^무` 매칭). 순수 그램/`g·oz`쌍 모두.
   - **(D) 헤더 깨진 컬럼형(워터블록)**: 무게행에 `g/oz`쌍, 헤더 사이즈가 OCR 로 깨짐 → **그램 값 개수로 사이즈 추론**(3→S/M/L).
   - **(E) 단일사이즈**: `단일사이즈` + `무게 3500 g` → 키 `""`.
   - **(F) 텐트 라벨형**: `최대 무게 : 1.55kg`(패킹) > `최소 무게` > `무게 : 230g`. **원단 무게(g/m²)는 `/m` 가드로 제외.**
7. **🔴 SIZE_HEAD 정규식은 긴 사이즈부터 나열.** `(?:M|MW)` 는 "MW"에서 M만 잡아 사이즈-무게가 전부 어긋난다
   (딥슬립 무게 전멸 원인). `LXW|MW|LW|SW` 를 `S|M|L|W` 보다 앞에. 대문자 `UNO/DUO` 도 `Uno/Duo` 와 별도 등재.
   (일반 룰이라 SKILL.md "엑스패드 세션"에도 등재.)
   - 남는 미추출: 폴라리스 텐트(OCR 이 가로표를 라벨/값으로 쪼갬), 코드 스터프색(3g, 바닥값 미만) — 오값 위험이라 방치.

### 색상 OCR (`exped-color-ocr.py`) — color 빈 제품 대상
8. 색상 없는 제품은 상세 이미지에 **`색상 : ● 메로우`** 형태로 색상이 그려져 있다(텍스트 아님).
   groupId 단위로 이미지 OCR → color(영문)+colorKorean(한글) 채움(73개 보강).
   - **마케팅 문장 오추출 필터 필수**: `색상과 디자인으로 업데이트 되었습니다` 류 → `되었|제공|업데이트|디자인|니다` + 길이>7 제외.
   - COLOR_EN 사전으로 영문화(메로우→Mallow, 모레인→Moraine 등). 멀티컬러는 첫 색상 대표.

### Cafe24 공통
9. 상세 에디터 이미지는 **`/web/upload/NNEditor/`** 경로(스펙·색상 OCR 대상). `src`/`ec-data-src`/`data-src` 모두 확인.
   Cafe24 서버렌더라 curl + 브라우저 UA 로 직접 fetch 가능.
